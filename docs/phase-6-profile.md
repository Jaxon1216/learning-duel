# Phase 6：用户信息卡片

## 目标

在右侧内容区底部显示用户的个人信息卡片。

## 组件：ProfileCard

显示内容：

- **姓名** — 加粗显示
- **学历** — 灰色小字
- **一句话简介** — 正常文字
- **联系方式** — 类型选择 + 值

### 联系方式的特殊设计

联系方式不是普通输入框，而是"类型 + 值"的组合：

```
[wx] [qq] [github] [email]   jiangxu05
```

- 4 个类型按钮并排
- 选中的类型是黑底白字（高亮）
- 未选中的是灰色
- 旁边显示联系方式的值

这样一眼就能看出这个人用什么方式联系。

### 编辑模式

- 看别人的卡片：纯展示
- 看自己的卡片：右上角有"编辑"按钮
- 点击"编辑"进入编辑模式（所有字段变成输入框）
- 点击"保存"提交到数据库
- 点击"取消"恢复原样

## 数据结构

```
profiles 表：
  user_id       → 主键，关联 auth.users
  name          → 姓名
  education     → 学历
  bio           → 一句话简介
  contact_type  → wx / qq / github / email
  contact_value → 联系方式值
```

## 权限

- 所有人能看到所有人的 profile（SELECT true）
- 只有自己能修改自己的 profile（UPDATE `auth.uid() = user_id`）

## 验收标准

- [x] 卡片显示姓名、学历、简介、联系方式
- [x] 联系方式类型有高亮效果
- [x] 自己的卡片可以编辑
- [x] 别人的卡片只读
- [x] 编辑后保存，刷新页面数据还在
